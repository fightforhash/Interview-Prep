## 문제 개요
두개의 singly LL 이 있을때, 하나의 sorted LL로 변환한다음 head를 돌려내시오, merged list는 새 리스트 노드 생성하지 말고 in place modification 하세요.

## 접근법
dummy 노드를 하나 생성하고, curr 포인터로 병합 리스트를 구성함.
두 리스트의 현재 노드 값을 비교하여 더 작은 것을 curr.next에 연결, 해당 리스트의 포인터도 한칸 이동시켜준다. 한쪽 리스트가 끝나면 남아 있는 리스트도 찾아서 연결한다.

## 시간/공간복잡도 
시간: O(N + M) N: list1의 노드 수 M: list2 의 노드 수 각각 한번씩 노드를 순회
공간: O(1) 포인터 사용
